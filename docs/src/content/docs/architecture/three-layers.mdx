## Overview

GoMode has three layers. Each can be understood independently.

```
CF Request → Worker (JS)
  → instantiates go.wasm (TinyGo + Zig ABI)
  → calls handle(reqPtr, reqLen)
  → Go processes request, uses Zig ABI for zero-copy ops
  → reads response from WASM memory
  → CF Response
```

## Layer 1: Zig ABI (`zig-abi/src/`)

Zig compiles to `wasm32-wasi` and exports functions with C ABI. Both Go and JS can call them directly — no serialization, just pointer math on shared WASM linear memory.

### Memory management

```
JS calls zig_alloc(100) → gets pointer 0x1A00 in wasm.memory
JS writes data at 0x1A00 → Go/Zig reads it at 0x1A00
Zero copy. Same memory, different languages reading it.
```

### Columnar tables

Data laid out column-by-column in WASM memory (Arrow-like):

```
ids:    [1, 2, 3, 4]           ← contiguous i32[] at ptr A
names:  "AliceBobCharlie"      ← contiguous bytes at ptr B
scores: [9.5, 8.2, 7.1, 9.9]  ← contiguous f64[] at ptr C
```

### Host imports

~10 functions in the `gomode_host` namespace: raw sockets, random bytes, time, console, KV. The JS worker implements these. All HTTP parsing, crypto, etc. stays inside WASM.

## Layer 2: CF Worker (`worker/src/`)

**`worker.ts`** — Entry point. Routes requests to direct WASM execution or a Durable Object.

**`go-do.ts`** — Durable Object. Instantiates WASM once, keeps it alive. Each request calls the exported `handle()` function.

**`columnar-proxy.ts`** — JS Proxy that reads columnar data from `wasm.memory` via DataView:

```typescript
const row = table.row(2);
row.id    // → DataView.getInt32(id_ptr + 2*4)
row.name  // → TextDecoder on the byte range
row.score // → DataView.getFloat64(score_ptr + 2*8)
```

## Layer 3: Go SDK (`go-sdk/`)

Go exports a `handle()` function that receives a pointer to request data in WASM memory and returns a pointer to response data.

```go
//export handle
func handle(reqPtr, reqLen uint32) uint32 {
  // Read request from shared memory
  // Process
  // Write response to shared memory
  // Return response length
}
```

Go calls Zig ABI functions via `//go:wasmimport`:

```go
//go:wasmimport gomode zig_alloc
func zigAlloc(len uint32) uint32

//go:wasmimport gomode zig_table_push_i32
func zigTablePushI32(table uint32, colIdx uint32, value int32) int32
```
