---
title: Worker vs Durable Object
description: When to use direct Worker mode vs Durable Object mode.
---

## Two modes

GoMode supports both stateless Workers and stateful Durable Objects. The Go handler code is identical — only the routing differs.

## Direct mode (Worker)

Requests go straight to WASM in the Worker isolate. CF spins up isolates as needed — each caches its own WASM instance.

```
Browser → CF Edge → Worker isolate
  → cached WASM instance
  → handle(reqPtr, reqLen)
  → Response
```

- **Stateless** — no shared state between requests (unless external KV/R2)
- **Max concurrency** — CF scales isolates horizontally
- **Cold start ~13ms** — compile WASM + init Go runtime, then cached
- **Route**: `/*`

## Durable Object mode

Requests route through a Durable Object. WASM instantiates once and stays alive for the DO's lifetime.

```
Browser → CF Edge → Worker → DO.fetch()
  → persistent WASM instance (stays alive)
  → handle(reqPtr, reqLen)
  → Response
```

- **Stateful** — in-memory state persists between requests
- **Single instance** — one DO globally, requests serialized
- **Zero cold start** after first request — WASM stays warm
- **Route**: `/do/*`

## Comparison

| | Direct (Worker) | Durable Object |
|---|---|---|
| State | None | In-memory + durable storage |
| Concurrency | CF scales isolates | Single instance, serialized |
| WASM lifetime | Cached per isolate, may evict | Alive for DO lifetime |
| Use case | APIs, transforms, auth | Sessions, counters, websockets, rate limiting |
| Cold start | ~13ms (then cached) | ~13ms (then persistent) |

## When to use which

**Use Direct mode** for stateless request/response handlers — API endpoints, data transforms, authentication checks, anything that doesn't need shared state.

**Use DO mode** when you need state that persists between requests — user sessions, real-time counters, websocket connections, rate limiters, collaborative editing.

You can mix both in the same worker:

```typescript
// worker.ts — already set up this way
if (url.pathname.startsWith("/do/")) {
  // Stateful → Durable Object
  return durable.fetch(request);
}
// Stateless → direct WASM
return handleDirect(request);
```
